《科技文档写作实务》

发表于 2018-03-06 | 分类于 读书记

《科技文档写作实务》一书由中兴通讯学院编著，一定程度上填补了国内对于技术文档撰写指导的空白。

产品需要全生命周期的管理，贯穿其中的是科技文档，依靠科技文档实现在不同环节的知识传递和产品价值链的耦合，在企业与市场间架起桥梁。

好的科技文档的一些要素：

  • 要求明确
  • 定义严谨
  • 指标具体
  • 方法直观
  • 简介规范
  • 善用图表
  • 易读而无歧义
  • 可操作
  • 可检验

概述篇

什么是技术文档

技术文档从属于科技文档，更偏重实践经验、操作方法的介绍，对实际生产生活有较强的知道意义。

技术文档的特征

  • 针对性：针对不同类型和层次的读者，文档内容和深度均不相同
  • 专业性：具备一定的专业背景
  • 功能性：以解决实际问题为目的
  • 规范性：统一的规范术语和缩略语

#47# 技术文档的多种形式

呈现形式：

  • 电子文档
  • 纸质文档
  • 多媒体文档
  • 联机帮助
  • 移动互联网文档

开发形式：

  • Word 开发形式
  • XML 开发形式
  • 图形图像开发形式（如AI、PS）
  • 多媒体开发形式

基础篇

技术文档写作流程：需求分析 → 架构设计 → 文档写作 → 质量控制 → → 文档发布 → 文档交付 → 文档维护

需求分析

信息收集

（1）收集读者信息

  • 读者的工作岗位
  • 读者的知识背景
  • 读者的阅读习惯

（2）收集文档需求信息

  • 了解原始需求
  • 文档使用的场景
  • 文档特殊需求
  • 交付时间点

（3）信息收集表

简化分类

（1）简化需求
（2）需求归类

  • 每类需求信息必须属于同一逻辑范畴
  • 分类方法不唯一
  • 分类要符合MECE（相互独立，完全穷尽）原则

分析需求

（1）过滤无效需求
（2）将需求转化为文档中需要落实的知识点
（3）明确需要收集的文档素材
（4）为文档写作提出写作要求

需求分析的方法
（1）“5W+1H分析法”
（2）继承法
（3）关联法
（4）借鉴法

确定开发任务：包括文档的范围和目的、读者对象、文档内容要求、文档开发计划、设备环境使用计划等。
