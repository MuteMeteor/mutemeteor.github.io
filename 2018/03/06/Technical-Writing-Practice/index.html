《科技文档写作实务》

发表于 2018-03-06 | 分类于 读书记

《科技文档写作实务》一书由中兴通讯学院编著，一定程度上填补了国内对于技术文档撰写指导的空白。

产品需要全生命周期的管理，贯穿其中的是科技文档，依靠科技文档实现在不同环节的知识传递和产品价值链的耦合，在企业与市场间架起桥梁。

好的科技文档的一些要素：

  • 要求明确
  • 定义严谨
  • 指标具体
  • 方法直观
  • 简介规范
  • 善用图表
  • 易读而无歧义
  • 可操作
  • 可检验

概述篇

什么是技术文档

技术文档从属于科技文档，更偏重实践经验、操作方法的介绍，对实际生产生活有较强的知道意义。

技术文档的特征

  • 针对性：针对不同类型和层次的读者，文档内容和深度均不相同
  • 专业性：具备一定的专业背景
  • 功能性：以解决实际问题为目的
  • 规范性：统一的规范术语和缩略语

技术文档的多种形式

呈现形式：

  • 电子文档
  • 纸质文档
  • 多媒体文档
  • 联机帮助
  • 移动互联网文档

开发形式：

  • Word 开发形式
  • XML 开发形式
  • 图形图像开发形式（如AI、PS）
  • 多媒体开发形式

基础篇

技术文档写作的流程：需求分析 → 架构设计 → 文档写作 → 质量控制 → → 文档发布 → 文档交付 → 文档维护

需求分析

信息收集

（1）收集读者信息

  • 读者的工作岗位
  • 读者的知识背景
  • 读者的阅读习惯

（2）收集文档需求信息

  • 了解原始需求
  • 文档使用的场景
  • 文档特殊需求
  • 交付时间点

（3）信息收集表

简化分类

（1）简化需求

（2）需求归类

  • 每类需求信息必须属于同一逻辑范畴
  • 分类方法不唯一
  • 分类要符合MECE（相互独立，完全穷尽）原则

分析需求

（1）过滤无效需求
（2）将需求转化为文档中需要落实的知识点
（3）明确需要收集的文档素材
（4）为文档写作提出写作要求

需求分析的方法
（1）“5W+1H分析法”
（2）继承法
（3）关联法
（4）借鉴法

确定开发任务

包括文档的范围和目的、读者对象、文档内容要求、文档开发计划、设备环境使用计划等。

架构设计

架构前的准备和学习

  • 产品介绍
  • 产品变更
  • 产品常用操作或维护
  • 产品相关技术
  • 目标读者遵从的国家标准、行业标准

设计架构

（1）构建知识地图

  • 分解主题
  • 确定主题类型
  • 确定主题的编写要求

（2）从知识地图产生文档架构

（3）架构检查

  • 评审对象：知识地图、各文档地图
  • 评审成员：作者、产品代表
  • 评审重点
  • 评审意见

文档写作

写作逻辑

（1）演绎逻辑和归纳逻辑
（2）归纳逻辑的逻辑顺序

  • 时间（步骤）顺序
  • 结构（空间）顺序
  • 程序顺序（重要性）顺序

写作6步法

  • 准备与学习
  • 确定主题写作目标
  • 分主题写作
  • 确定逻辑顺序
  • 快速写作
  • 不断完善

写作黄金法则

  • 主题明确单一
  • 语言简洁明了
  • 图表优于文字
  • 例子增强理解

质量控制

  • 文档审核
  • 文档测试
  • 文档验证

进阶篇

实用性

  • 安全第一
  • 内容准确
  • 面向任务
  • 组织合理
  • 内容完整

可读性

易读

（1） 采用恰当的词汇

  • 突出含义，简洁易懂
    • 避免使用介词、连接词、转折词
    • 避免使用表示程度、强调语气的副词
    • 避免使用代词，请给出完整的称呼
  • 避免歧义，明确是否
    • 使用含义明确的词语
    • 使用正确的修饰词
    • 注意容易混淆的词汇
    • 涉及时间的时候，用具体的年月日来表示
  • 规范用语，称呼一致
    • 采用规范书面用语
    • 注意词汇、术语的前后统一
  • 计量单位国际标准

（2） 采用语义明确的句子

  • 语法准确，动词恰当
  • 句子完整，不长不短
  • 尊重读者，尊重文化

（3） 保证段落清晰

  • 善用列表
  • 善用表格
  • 善用图形

（4） 遵循模板和样式

易理解

  • 举例增强理解
  • 术语和缩略语准确定义
  • 标题反映主题思想
  • 概述有机联系上下文
  • Step by step

舒适性

  • 文档获取便捷
  • 内容检索高效
  • 阅读感受舒适
