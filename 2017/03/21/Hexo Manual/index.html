Hexo Manual

2017.03.21 Mute Tech


I have moved from Farbox to Hexo for several days. Though I run into troubles installing Hexo and solved it for almost a day, I felt joyful when I finished establishing this page. That’s awfully fulfilling.

Just as Hexo says, it is a fast, simple & powerful blog framework.

We can write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

Compared with jekyll which is framed on Ruby, Hexo is faster and easier with Node.js.

This post is written for myself to remember the main operations of Hexo and git.

Git

Workflow

The local repository consists of three “trees” maintained by git.

The first one is your Working Directory which holds the actual files.
The second one is the Index which acts as a staging area and finally the HEAD which points to the last commit you’ve made.
